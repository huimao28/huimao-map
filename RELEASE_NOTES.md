# 灰猫地图 1.0.21 更新说明

## 修复内容

- 修复长距离导航刚启动或进入沿途推荐面板时闪退的问题。
- 根因是百度导航 SDK 内部引用 `com.bumptech.glide.Glide`，但离线 AAR 没有携带该传递依赖；现在主应用显式打包 Glide 4.16.0。
- 设置页和百度 AK 提示中的 Release SHA-1 已统一为固定长期签名：`DE:FF:00:C2:E1:8A:20:62:9C:4A:17:67:B7:27:A5:08:CC:1B:F0:E4`。
- Android Auto 指引同步的延迟读取增加异常隔离；百度导航面板结构异常时只跳过同步，不再影响手机导航线程。

## 未修改内容

- 未修改路线规划、导航定位、Android Auto 地图、比例尺和坐标处理。
- 未修改手机百度导航 View 的布局和主题。
- 三个微信位置转发插件仅随版本重新构建，转发逻辑未修改。

## 已知问题

- 百度 SDK 可能还有其他未声明的可选传递依赖，若设备日志出现新的 `NoClassDefFoundError`，将按实际类名补齐并单独发布修复。
- Android Auto 地图仍固定北向上，局部瓦片下载失败时可能显示深色背景。

## 安装说明

- 主应用 `versionCode` 为 22，使用固定长期 release 证书，可直接覆盖安装 1.0.20。
- 百度地图 AK 应在百度控制台绑定包名 `com.huimao.map` 和上述 Release SHA-1。
- Release 同时提供百度、高德和腾讯三个微信位置转发插件；只需安装一个与现有地图应用不冲突的版本。
